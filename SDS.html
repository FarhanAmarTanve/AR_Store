1. Introduction

This design document describes all the data, architectural, interface and component level designs for the Smart Furniture System. The design specification provides explicit information about the requirements for the product and how the product will be put together.

 

2. Document Conventions

Typical document conventions are maintained where main topics are bold, have a font size of about 20pt. Sub topics are bold, have a smaller font size of about 16pt and are numbered with the main topic number as a prefix, e.g. 1.1 to 1.5.

 

3. Intended Audience and Reading Suggestion

The intended audience for this document is the project managers and developers. This document is meant to be used as a guideline for all the development needs for its creation. The document contains several technical terminologies and diagrams; hence readers are advised to have predetermined knowledge about the topics covered.

 

4. Description of Design Components

We will be giving a basic overview of the various components to be used in our system. SQLite will be used as our database software for Hand held software application and PostgreSQL will be used for our web applications. To represent various data from our project, we will be using tables included within the database software. We will be provided with an interface for us to request information or submit updates to the server through the server procedure proxy. The clients (Manufacturer) will be given limited access to the server so that they can update the contents

 

5. Class Diagrams

This section focuses on the high-level class diagram which demonstrates the relation between the classes. A detailed description of each class is given for a clear grasp of their function in the product.

 

6. ER Diagrams

Entity relationship diagram displays the relationships of entity set stored in a database. In other words, we can say that ER diagrams help you to explain the logical structure of databases.
